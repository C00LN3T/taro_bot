# Чек-лист разработки бота «Гадалка»

## Подготовка
- [x] Заполнить `.env` с `BOT_TOKEN`, `ADMIN_IDS`, `DEFAULT_LANGUAGE`.
- [x] Установить зависимости: `pip install -r requirements.txt`.
- [x] Настроить хранилище данных (SQLite/PostgreSQL) и миграции/инициализацию.

## Пользовательский сценарий
- [x] Приветствие и главное меню с кнопками.
- [x] FSM для всех типов раскладов (таро, нумерология, астрология) с запросом входных данных.
- [x] Возможность отмены/возврата в меню из любого состояния.
- [x] Профиль пользователя: отображение и редактирование даты рождения/языка/имени/пола, удаление данных.

## Расклады и расчёты
- [x] Минимальный таро-расклад (3 карты) и совет дня.
- [x] Базовый расчёт числа судьбы.
- [x] Расчёт числа имени и краткой нумерологической карты, совместимость дат.
- [x] Определение знака зодиака и краткое описание.
- [x] Случайный выбор системы расклада с учётом разрешений/лимитов.
- [x] Подключение полноценных справочников карт/чисел/знаков из БД.

## Безопасность и качество
- [x] Логирование входящих событий и ошибок.
- [x] Лимиты и (опционально) платёжные сценарии (лимиты реализованы, платежи — точка расширения).
- [x] Тестовые сценарии из ТЗ: корректный/некорректный ввод дат, быстрые запросы, возврат в меню (unit-тесты сервисов + ручные сценарии).

## Администрирование
- [x] Команды `/admin_stats` и `/broadcast` с проверкой списка админов.
- [x] Статистика по пользователям и раскладам.
